# Credit Card Fraud Detection

Detecting fraudulent credit card transactions using a three-part approach: supervised learning, unsupervised anomaly detection, and cost-sensitive evaluation — built to compare methodologies rather than rely on a single model and metric.

## Dataset

European cardholder transactions from September 2013, sourced from the [ULB Credit Card Fraud dataset](https://www.kaggle.com/mlg-ulb/creditcardfraud). 284,807 transactions, 492 labeled as fraud (0.172%). Features V1–V28 are PCA-transformed for confidentiality; only `Time` and `Amount` remain in their original form.

The raw dataset is not included in this repo due to size. Download `creditcard.csv` from the link above and place it in `data/raw/`.

## Why this structure

Standard fraud detection portfolios use accuracy as a metric (meaningless here, since predicting "genuine" for everything scores 99.8%) and treat every misclassification as equally costly (also false — missing a $500 fraud is not the same as flagging one genuine transaction for review). This project addresses both problems directly:

1. A supervised model that handles the class imbalance properly and is evaluated on AUPRC, not accuracy.
2. An unsupervised anomaly detection model, representing a realistic scenario where fraud labels aren't yet available.
3. A cost-sensitive evaluation that scores both models by actual dollar impact, not abstract metrics alone.

## Findings so far

**EDA (`01_eda.ipynb`)** — Confirmed severe class imbalance (0.172% fraud), found and removed 1,081 duplicate rows, identified V14, V17, V12, V10, V11, V4 as the strongest fraud-linked features via correlation. Saved a fixed, stratified 80/20 train/test split used consistently across all later notebooks.

**Supervised model (`02_supervised_model.ipynb`)** — XGBoost trained with `scale_pos_weight` (599.48) to correct for class imbalance, no resampling (SMOTE) used. Achieved **AUPRC 0.8139**. At default threshold: 79% recall, 91% precision, catching 75 of 95 fraud cases. Feature importance showed V14 as the dominant predictor (52.8% of total importance) — notably, V17, the strongest *linear* correlate with fraud, ranked low in the model's actual importance, confirmed not to be due to redundancy with V14 (correlation between the two: -0.011).

**Anomaly detection (`03_anomaly_detection.ipynb`)** — Isolation Forest trained without fraud labels, using the true contamination rate. Achieved **AUPRC 0.1040** — tested against two different contamination settings with identical results, confirming this is a genuine limitation of unsupervised detection on this dataset, not a tuning issue. At default threshold: only 19% recall, 20% precision. This result is intentional, not a failure — it quantifies, with real numbers, why labeled data matters when it's available, and establishes a realistic comparison point for scenarios where labels are delayed.

**Cost-sensitive evaluation (`04_cost_sensitive_eval.ipynb`)** — in progress. Will evaluate both models against actual dollar cost (false negative cost = missed fraud amount, false positive cost = fixed review cost), rather than AUPRC alone.
